# 模型推理参数配置指南

## 🎯 参数选择理由

### **Temperature（温度）**
控制输出的"创意度"：
- **低（0.08-0.12）**：确定性强，适合保守任务（文本压缩）
- **中（0.25）**：平衡创意和稳定
- **高（0.70）**：高创意性，适合开放式推理

**本项目选择：**
- Gemma-4-E4B (0.10)：基础模型，需要稳定
- Gemma-4-E2B (0.08)：**最保守** - 轻量级模型容易不稳定
- GLM-Edge-4B (0.12)：**稍高** - 中文生成需要灵活性
- Qwen-3.5-4B (0.70)：**官方推荐** - 微调用途是推理
- Phi-4-Mini (0.25)：**中等** - 工具调用需要平衡

---

### **Top-P（核采样）**
控制候选词汇的累积概率：
- **低（0.80-0.85）**：严格，只选最可能的词
- **中（0.88-0.92）**：平衡
- **高（0.95+）**：开放，允许长尾词

**本项目选择：**
- Gemma-4-E2B (0.85)：**最严格** - 轻量级需要约束
- Qwen-3.5-4B (0.80)：**严格** - 官方推荐
- Phi-4-Mini (0.88)：**中等** - 平衡
- Gemma-4-E4B (0.90)：**中等** - 基础模型
- GLM-Edge-4B (0.92)：**最开放** - 中文需要多样性

---

### **Top-K（K采样）**
限制候选词汇的数量：
- **低（20-32）**：只考虑前 K 个最可能的词
- **中（35-45）**：平衡覆盖和确定性
- **高（50+）**：大量候选词

**本项目选择：**
- Qwen-3.5-4B (20)：**最严格** - 官方推荐
- Phi-4-Mini (32)：**保守** - 防止工具指令触发
- Gemma-4-E2B (35)：**严格** - 轻量级
- Gemma-4-E4B (40)：**中等** - 标准配置
- GLM-Edge-4B (45)：**最开放** - 中文多样性

---

### **Repeat Penalty（重复惩罚）**
防止模型重复生成相同内容：
- **低（1.0）**：允许重复（Qwen官方推荐）
- **中（1.08-1.15）**：适度惩罚
- **高（1.20+）**：严格防重复

**本项目选择：**
- Qwen-3.5-4B (1.00)：**无惩罚** - 官方推荐，优先级最高
- GLM-Edge-4B (1.08)：**最小** - 中文自然表达需要重复
- Gemma-4-E4B (1.10)：**标准** - 基础配置
- Gemma-4-E2B (1.15)：**高** - 轻量级易陷入循环
- Phi-4-Mini (1.20)：**最高** - 防止工具指令重复执行

---

## 🔬 模型微调方向分析

### Gemma-4-E4B / E2B
- **微调目标**：通用对话
- **推荐策略**：**保守+稳定**
- 为什么不一样：E2B 是轻量级版本，容易陷入不稳定状态

### GLM-Edge-4B
- **微调目标**：中文优化、边界应用
- **推荐策略**：**开放+多样**
- 为什么不一样：中文表达需要更多样化采样

### Qwen-3.5-4B
- **微调目标**：CoT 推理、复杂推理
- **推荐策略**：**官方参数优先**
- 为什么不一样：官方文档详细，已验证有效

### Phi-4-Mini
- **微调目标**：工具调用、指令跟随
- **推荐策略**：**平衡+防护**
- 为什么不一样：容易执行不必要的工具指令，需要约束

## 🎯 压缩任务优化思路

虽然这些模型被微调用于不同目标，但在压缩任务中我们的策略是：

```
1. 保守的 Temperature（0.08-0.12）
   ↓
   强制模型输出确定性内容，不创意

2. 严格的 Top-K 采样（20-45）
   ↓
   限制选词范围，避免离题

3. 定制的 Repeat Penalty
   ↓
   根据模型特性防止特定问题

4. 平衡的 Top-P
   ↓
   在保守和多样性间找到平衡点
```

## 💡 使用这些参数的方式

### **方式 1：直接在 Ollama 命令行**
```bash
ollama run gemma4-e4b-compress \
  "你的压缩任务" \
  temperature 0.1 \
  top_p 0.9 \
  top_k 40
```

### **方式 2：使用 Python 客户端**
```python
from config import get_model_gen_args

model_args = get_model_gen_args("gemma4-e4b")
result = client.infer(
    "gemma4-e4b",
    text,
    system_prompt=system_prompt,
    **model_args  # 自动应用最优参数
)
```

### **方式 3：创建 Modelfile**（本脚本方式）
```
FROM ./model.gguf
PARAMETER temperature 0.1
PARAMETER top_p 0.9
PARAMETER top_k 40
```

## 📈 参数调优建议

如果需要进一步调优：

1. **提高压缩率**：降低 temperature 和 top_k
2. **提高多样性**：提高 temperature 和 top_p
3. **防止重复**：增加 repeat_penalty
4. **防止离题**：降低 top_k
